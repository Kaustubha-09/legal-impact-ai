# LifeLaw

> A personalized legal intelligence platform. Give it your state, city, county, and profile tags (tenant, gig worker, H-1B, etc.) and it builds a filtered feed of live federal and state bills, a rights library, court case summaries, and a source-grounded Q&A answer engine — no legal advice, no law firm, just educational context tied to your actual jurisdiction.

Most "legal AI" products either paste raw statute text at you or let an LLM freewheel an answer. LifeLaw takes a narrower, more honest path: retrieval is real (pgvector cosine search over locally-embedded rights topics, statutes, and live bills), synthesis is structured (answers are assembled from retrieved fields, not generated free-text — no hallucination surface), and jurisdiction is load-bearing (every feed item, bill, and rights topic is filtered to what actually applies to *your* state/city/county).

## Features

### Personalized feed
- **Live federal bills** — Congress.gov `/v3/bill`, sorted by most recently updated, auto-tagged against profile categories (Tenant, Employee, H-1B, Gig worker, etc.)
- **Live state bills** — LegiScan `getMasterList`, covering all 50 states + DC, sorted by last action date
- **Radar** — new items since your last visit are badged and can be marked-as-read in bulk
- **Jurisdiction-coded UI** — federal / state / court / local badges use distinct design-token colors so scope is visible at a glance

### Ask AI (RAG, no hallucination)
- Free local `sentence-transformers` embeddings (`all-MiniLM-L6-v2`, 384-dim) — no LLM API key required
- pgvector cosine-distance retrieval over a curated corpus of rights topics and legal documents
- Answers are assembled from retrieved structured fields, not generated by an LLM — if nothing matches above the similarity threshold, it says so instead of guessing
- Regex-based fast paths for common patterns ("I got fired," "police searched my phone") before falling back to retrieval

### Rights library, search, and case law
- 18 rights topics with laws, cases, common questions, and state-specific notes
- Keyword/jurisdiction/source-type filtered search over a 13-document legal index
- Full case write-ups (issue, holding, reasoning, rights affected, plain-English explanation) for landmark rulings like *Riley v. California* and *Carpenter v. United States*

### Weekly email digest
- Opt-in email field on the anonymous device profile — no account system required
- `python -m app.services.digest` computes new feed items per profile since their last send and emails via Resend
- One-click unsubscribe endpoint; per-recipient send failures don't block the rest of the batch
- Untrusted upstream bill titles/summaries are HTML-escaped before landing in an email

## Setup

### Frontend

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

### Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

Requires PostgreSQL with the `pgvector` extension and a `backend/.env` with `DATABASE_URL`, `CONGRESS_API_KEY`, `LEGISCAN_API_KEY`, and `RESEND_API_KEY` (all free-tier signups). Run `python -m app.services.ingest` once to embed the curated corpus into `legal_sources`.

### Weekly digest

```bash
cd backend
python -m app.services.digest
```

Intended to run on a schedule (cron/Task Scheduler) once a Resend sending domain is verified.

## Design decisions worth knowing

- **No LLM call in the Q&A path.** Retrieval-only synthesis was chosen deliberately over generative synthesis to eliminate the hallucination surface — every answer is traceable to a specific retrieved document.
- **Anonymous device-ID identity.** No auth system yet; a `crypto.randomUUID()` generated client-side and persisted in `localStorage` stands in for a user ID, reused as the `UserProfile.clerk_user_id` column so a future Clerk migration doesn't require a schema change.
- **Per-source failure isolation.** `/api/feed` wraps each external API call (Congress.gov, LegiScan) in its own try/except — one dead upstream API degrades the feed instead of breaking it.
